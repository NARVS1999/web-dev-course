Lesson 0001: HTML Foundations

Ang pundasyon ng bawat website — simulan natin dito!

Lesson Goal Sa araling ito, matututunan mo ang basic na structure ng HTML at makakagawa ka ng sarili mong web page — isang simple portfolio page na pwedeng ipakita sa potential clients!

Ano ang HTML?

Ang HTML (HyperText Markup Language) ang istruktura ng bawat website. Isipin mo na lang na ang website ay isang bahay:

  • HTML = ang mga semento, haligi, at dingding — yung structure
  • CSS = ang pintura, disenyo, at dekorasyon — pampaganda
  • JavaScript = ang kuryente at plumbing — nagpapagalaw at nagpapa-function

Sa lesson na ito, HTML lang muna ang focus natin. Sa susunod na lessons, dadagdagan natin ng CSS para gumanda, at JavaScript para gumalaw!

Structure ng isang HTML Document

Eto ang pinaka-basic na HTML document:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Page</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <p>This is my first web page.</p>
</body>
</html>

Ano ang ibig sabihin ng bawat parte?

Tag Kahulugan Gamit
<!DOCTYPE html> Document type declaration Sinasabi nito sa browser na HTML5 ang gamit mo
<html> Root element Ang pinaka-parent ng lahat ng elements
<head> Metadata container Dito nilalagay ang title, styles, at ibang impormasyon na hindi visible sa page
<body> Content container Dito nilalagay ang lahat ng nakikita sa page
<h1> to <h6> Headings Mga heading/subheading — h1 ang pinakamalaki, h6 ang pinakamaliit
<p> Paragraph Paragraph text
Tip Pansinin na halos lahat ng HTML tags ay may opening tag (<tag>) at closing tag (</tag>). May "/" ang closing tag!

Mga Common HTML Tags na Dapat Mong Malaman

Text Tags

<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Sub-subheading</h3>
<p>Ito ay isang paragraph ng text. Pwedeng mahaba o maiksi.</p>
<p>Ito ay <strong>bold text</strong> at ito ay <em>italic text</em>.</p>

Links at Images

<!-- Link -->
<a href="https://google.com">Pumunta sa Google</a>

<!-- Image -->
<img src="https://example.com/photo.jpg" alt="Description ng photo">
Pansinin Ang <img> ay self-closing tag — walang closing tag! At ang <a> naman ay may href attribute na nagsasabi kung saan mapupunta ang link.

Lists

Unordered List (may bullet points)

<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
</ul>

Ordered List (may numbers)

<ol>
  <li>Unang hakbang</li>
  <li>Ikalawang hakbang</li>
  <li>Ikatlong hakbang</li>
</ol>

Forms (important to para sa freelancing!)

<form>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">

  <label for="email">Email:</label>
  <input type="email" id="email" name="email">

  <button type="submit">Send</button>
</form>

Div at Span — Ang mga Container

<div>Ito ay isang block-level container. Mag-isa sa linya.</div>
<span>Ito ay inline container. Sumasabay sa text.</span>
Block vs Inline Block elements (like <div>, <p>, <h1>) — mag-isa sa linya, full width. Inline elements (like <span>, <a>, <strong>) — sumasabay sa text, hindi nag-break ng linya.

Semantic HTML — Why It Matters for Freelancing

Ang semantic HTML ay ang paggamit ng tamang tag para sa tamang purpose. Halimbawa, sa halip na <div> lang ang gamitin sa lahat, may mas meaningful na tags: 

<header>Site Header / Navigation</header>
<nav>Navigation Menu</nav>
<main>Main Content</main>
<section>A Section of Content</section>
<article>A Blog Post or Article</article>
<aside>Sidebar / Related Content</aside>
<footer>Footer / Copyright</footer>
Bakit important ito sa freelancing? Mas gusto ng Google ang semantic HTML (mas maganda ang SEO — Search Engine Optimization). At mas madali i-maintain ng ibang devs ang code mo. Ibig sabihin: mas mataas ang value ng services mo sa clients!

🛠️ Mini-Project: Your First Portfolio Page

Gawin Mo Ito!

Gumawa ng HTML file na portfolio.html sa desktop mo. I-save at i-open sa browser.

1

Gumawa ng bagong file na portfolio.html

2

Kopyahin itong code:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Portfolio</title>
</head>
<body>
  <header>
    <h1>Juan Dela Cruz</h1>
    <p>Web Developer | Freelancer</p>
  </header>

  <main>
    <section>
      <h2>About Me</h2>
      <p>I'm learning web development para makapag-start ng freelancing career. I love building things that help people.</p>
    </section>

    <section>
      <h2>My Skills</h2>
      <ul>
        <li>HTML (kakasimula palang!)</li>
        <li>CSS (soon!)</li>
        <li>JavaScript (next!)</li>
      </ul>
    </section>

    <section>
      <h2>Contact Me</h2>
      <p>Email: <a href="mailto:juan@example.com">juan@example.com</a></p>
    </section>
  </main>

  <footer>
    <p>&copy; 2026 Juan Dela Cruz</p>
  </footer>
</body>
</html>
3

I-open sa browser! Double-click ang portfolio.html file — makikita mo na ang unang website mo! 🎉

🎯 Lesson Summary

  • Ang HTML ang nagbibigay ng structure sa website (parang skeleton ng tao)
  • Ang HTML document ay may <head> (metadata) at <body> (visible content)
  • May iba't ibang tags: headings, paragraphs, links, images, lists, at forms
  • Ang semantic HTML ay important para sa SEO at maintainability — mas gusto ng clients 'to
  • Nakagawa ka na ng unang portfolio page mo! 🎉
