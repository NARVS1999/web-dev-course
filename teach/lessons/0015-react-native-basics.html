Lesson 0015: React Native Basics

Gawing mobile app ang iyong React skills — isang codebase, parehong iOS at Android!

Lesson Goal Matutunan ang pundasyon ng React Native gamit ang Expo: setup, core components, styling, at Flexbox — at makagawa ng isang working counter app sa iyong phone.

Part 1: Ano ang React Native?

Ang React Native ay isang framework na gumagamit ng React para gumawa ng native mobile apps (iOS at Android). Hindi ito WebView o hybrid app — native components talaga ang ginagamit.

React vs React Native Sa React (web), gumagamit tayo ng <div>, <span>, <h1>. Sa React Native naman, <View>, <Text>, atbp. Pareho pa rin ang konsepto ng components, props, at state!

Part 2: Setup with Expo

Ang Expo ay isang toolchain na nagpapadali ng React Native development — no need para sa Xcode (iOS) o Android Studio para magsimula.

1

I-install ang Node.js — kung wala ka pa, nasa nodejs.org (20.x LTS version).

2

Install Expo CLI at gumawa ng project (Expo SDK 52):

npx create-expo-app@latest MyMobileApp
cd MyMobileApp
3

I-download ang "Expo Go" app sa iyong phone (Play Store o App Store).

4

Run the project:

npx expo start

May lalabas na QR code sa terminal. I-scan ito gamit ang Expo Go app sa iyong phone — lilitaw agad ang iyong app!

Part 3: Core Components

Sa React Native, ang basic building blocks ay iba sa web:

React (Web)React NativePurpose
<div><View>Container / layout
<span>, <p><Text>Text display
<input><TextInput>Text input field
<button><Button> / <TouchableOpacity>Pressable element
<img><Image>Display images
<ul>/<li><FlatList> / <ScrollView>Scrollable lists

Halimbawa ng isang simpleng screen:

// App.js
import { View, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello, Mobile!</Text>
      <Text style={styles.subtitle}>My first React Native app</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#333',
  },
  subtitle: {
    fontSize: 16,
    color: '#666',
    marginTop: 8,
  },
});

Part 4: Styling in React Native

Walang CSS sa React Native — lahat ay JavaScript objects gamit ang StyleSheet.create():

  • Pareho pa rin ang properties: backgroundColor, fontSize, padding, margin
  • Ang properties ay camelCase — background-color → backgroundColor
  • Ang layout ay Flexbox by default — display: flex automatic
  • Ang flexDirection ay column (hindi row tulad ng web)

Part 5: State & Events

Gumagana ang useState at useEffect sa React Native tulad ng sa web:

import { useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <View style={styles.container}>
      <Text style={styles.count}>{count}</Text>
      <TouchableOpacity
        style={styles.button}
        onPress={() => setCount(count + 1)}
      >
        <Text style={styles.buttonText}>+1</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  count: { fontSize: 48, fontWeight: 'bold', marginBottom: 20 },
  button: {
    backgroundColor: '#2563eb',
    paddingHorizontal: 32,
    paddingVertical: 12,
    borderRadius: 8,
  },
  buttonText: { color: '#fff', fontSize: 18, fontWeight: '600' },
});
TouchableOpacity vs Button Ang TouchableOpacity ay mas flexible — pwedeng maglaman ng kahit anong component sa loob, may opacity feedback kapag pinindot. Ang Button naman ay limited sa text lang at may sariling style.
