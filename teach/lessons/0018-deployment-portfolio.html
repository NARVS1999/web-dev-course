Lesson 0018: Deployment & Portfolio

I-deploy ang iyong Laravel API sa shared hosting at buuin ang iyong portfolio para sa freelancing.

Lesson Goal Matutunan kung paano i-deploy ang Laravel API sa shared hosting (cPanel), i-setup ang production environment, at bumuo ng portfolio na magpapakita ng lahat ng iyong projects.

Part 1: Deployment Options

Para sa freelance projects, may iba't ibang paraan para i-deploy ang iyong Laravel API:

PlatformCostGood For
Shared hosting (cPanel)~$5-10/monthSmall projects, beginners
VPS (DigitalOcean, Linode)~$6-12/monthMedium projects, more control
Render / RailwayFree tier availableSimple APIs, prototypes
Shared hosting tip Maraming shared hosting providers (Hostinger, Namecheap, SiteGround) ang may isa-click na Laravel installer sa cPanel. I-check ang "Softaculous" o "Installatron" sa cPanel dashboard.

Part 2: Deploy Laravel to Shared Hosting (cPanel)

1

Maghanda ng production database: Sa cPanel, gamitin ang MySQL Wizard para gumawa ng database at user. I-note ang credentials.

2

I-update ang .env para sa production:

APP_ENV=production
APP_DEBUG=false
APP_URL=https://yourdomain.com

DB_HOST=localhost
DB_DATABASE=your_database_name
DB_USERNAME=your_database_user
DB_PASSWORD=your_database_password
3

I-upload ang files via FTP o File Manager: I-upload ang buong Laravel project sa public_html/ (o subfolder).

4

Ilipat ang contents ng public/ sa root: Sa shared hosting, ang document root ay nakaturo sa public_html/. Kailangan mong ilipat ang laman ng public/ sa public_html/ at i-update ang index.php paths.

Mas madaling alternatibo: gumamit ng .htaccess para i-rewrite ang requests:

# public_html/.htaccess
RewriteEngine On
RewriteCond %{REQUEST_URI} !^/public/
RewriteRule ^(.*)$ public/$1 [L]
5

Run migrations: I-access ang iyong site at magdagdag ng temporary route para i-run ang migrations, o gamitin ang cPanel terminal:

php artisan migrate --force
Security Reminder Siguraduhing i-delete ang installation routes pagkatapos ng migrations. I-set ang APP_DEBUG=false para hindi mag-leak ng sensitive info.

Part 3: Building Your Portfolio

Ngayong marami ka nang projects, kailangan mo ng portfolio site para ma-present ang mga ito sa clients.

Ano ang Dapat Laman ng Portfolio:

  • Home / Hero Section — Maikling intro kung sino ka at ano ang kaya mong gawin
  • Services — Anong klase ng websites ang kaya mong gawin (portfolio, e-commerce, blog, atbp.)
  • Projects — Listahan ng iyong mga nagawang projects na may screenshots at description
  • Skills — HTML, CSS, JS, Laravel, Next.js, React Native, MySQL
  • Contact — Form o direct contact info para maabot ka ng clients

Gamitin ang Iyong Next.js Portfolio

I-deploy ang iyong Next.js portfolio sa Vercel (gaya ng natutunan sa Lesson 0014) at i-customize ito para sa freelancing:

// pages/projects.js — Projects page
import Link from 'next/link';

const projects = [
  {
    title: 'Blog API',
    desc: 'Laravel REST API with authentication and CRUD operations.',
    tech: ['Laravel', 'MySQL', 'Sanctum'],
  },
  {
    title: 'Portfolio App',
    desc: 'Multi-page Next.js portfolio with SSR and optimized images.',
    tech: ['Next.js', 'Vercel'],
  },
  {
    title: 'Mobile Blog Reader',
    desc: 'React Native app connected to Laravel API with offline caching.',
    tech: ['React Native', 'Expo', 'AsyncStorage'],
  },
];

export default function Projects() {
  return (
    <div>
      <h1>My Projects</h1>
      {projects.map((p, i) => (
        <div key={i} style={{ marginBottom: 24 }}>
          <h2>{p.title}</h2>
          <p>{p.desc}</p>
          <p><strong>Tech:</strong> {p.tech.join(', ')}</p>
        </div>
      ))}
    </div>
  );
}

Mini-Project: Deploy & Portfolio

Gawin ang sumusunod:

  • I-deploy ang iyong Laravel Blog API sa shared hosting (o Render free tier)
  • I-update ang inyong Next.js portfolio para mag-include ng projects page
  • I-deploy ang portfolio sa Vercel
  • I-connect ang portfolio sa deployed API (i-update ang NEXT_PUBLIC_API_URL)
  • I-share ang link sa portfolio — handa ka nang mag-freelance!
