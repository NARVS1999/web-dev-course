Lesson 0014: Next.js Deployment

I-deploy ang iyong Next.js portfolio sa Vercel — libre at walang sakit ng ulo.

Lesson Goal Matutunan kung paano i-deploy ang Next.js app sa Vercel, mag-setup ng environment variables, magdagdag ng custom domain, at i-connect ang frontend sa iyong Laravel backend.
Tech Stack Itinuturo ng lesson na ito ang Next.js 15.x gamit ang Pages Router. Tiyaking ang iyong project ay gumagamit ng bersyong ito mula sa Lesson 0011.

Part 1: Why Vercel?

Ang Vercel ay ang company na gumawa ng Next.js — kaya sila ang may pinakamahusay na deployment experience para dito. Libre ang hobby plan at may kasamang:

  • Automatic HTTPS/SSL
  • Global CDN (100+ locations)
  • Automatic preview deployments sa bawat git push
  • Serverless functions para sa API Routes
  • Custom domains
Alternatives Pwede ring i-deploy sa Netlify (gamit ang @netlify/plugin-nextjs), Cloudflare Pages, o sa traditional hosting gamit ang next build && next start.

Part 2: Preparation

Bago mag-deploy, siguraduhing handa na ang iyong project:

1

Push to GitHub (or GitLab/Bitbucket):

git init
git add .
git commit -m "Initial commit: Next.js portfolio"
git remote add origin https://github.com/your-username/my-portfolio.git
git push -u origin main
2

Check your next.config.js — dapat may images config kung gagamit ka ng external images:

// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    domains: ['localhost', 'your-laravel-api.com'],
  },
};

module.exports = nextConfig;
3

I-update ang API URLs — sa development, localhost:8000 ang gamit. Sa production, kailangan ng actual URL ng iyong deployed Laravel API:

# .env.local (local development)
NEXT_PUBLIC_API_URL=http://localhost:8000/api

# .env.production (ito ang gagamitin sa Vercel)
NEXT_PUBLIC_API_URL=https://your-laravel-api.com/api

Sa Vercel, ise-set mo ang environment variables sa dashboard — hindi mo kailangan ng .env.production file.

Part 3: Deploy to Vercel

1

Create a Vercel account: Pumunta sa vercel.com at mag-sign up gamit ang GitHub account.

2

Import your repository:

  • Click "Add New" → "Project"
  • Piliin ang my-portfolio repository
  • I-configure ang project settings

3

Add environment variables: Sa Vercel dashboard, mag-add ng:

NEXT_PUBLIC_API_URL = https://your-laravel-api.com/api

Kung may API token, idagdag din ito bilang environment variable (walang NEXT_PUBLIC_ prefix para secure).

4

Click "Deploy": Awtomatikong magbu-build at magde-deploy ang Vercel. Pagkatapos ng ilang minuto, makikita mo ang URL ng iyong site (hal. my-portfolio.vercel.app).

Continuous Deployment Tuwing magpi-push ka ng changes sa GitHub, awtomatikong nagde-deploy ang Vercel. May Preview URLs din para sa bawat branch — perfect para sa testing bago i-merge sa main.

Part 4: Custom Domain

Gawing propesyonal ang iyong portfolio gamit ang sarili mong domain:

  1. Bumili ng domain mula sa Namecheap, GoDaddy, o Google Domains
  2. Sa Vercel dashboard, pumunta sa project → "Domains"
  3. I-type ang iyong domain at i-click "Add"
  4. I-update ang DNS settings ng iyong domain:
    • Add CNAME record: www → cname.vercel-dns.com
    • Add A records: ituro ang apex domain (@) sa Vercel's IP addresses
  5. Hintaying mag-propagate ang DNS (mga ilang minuto hanggang 48 oras)

Vercel will automatically provision an SSL certificate para sa iyong domain — libre!

Part 5: Connecting to Laravel Backend

Para sa production, kailangan mo ring i-deploy ang iyong Laravel API. Pwedeng gamitin ang:

  • Shared hosting (cPanel) — cheapest option (~$5-10/month)
  • DigitalOcean / Linode — VPS (~$6-12/month)
  • Laravel Forge — managed server (~$15/month)

Kapag naka-deploy na ang Laravel API, i-update ang NEXT_PUBLIC_API_URL sa Vercel dashboard:

NEXT_PUBLIC_API_URL = https://api.yourdomain.com
CORS Siguraduhing may CORS configuration ang iyong Laravel backend para payagan ang requests mula sa iyong Next.js domain. Gamitin ang config/cors.php o laravel-cors package.

'allowed_origins' => ['https://my-portfolio.vercel.app']

Final Project: Deploy Your Portfolio

Ilive ang iyong buong portfolio:

  • I-push ang iyong Next.js project sa GitHub
  • I-deploy sa Vercel (import project mula sa GitHub)
  • Magdagdag ng NEXT_PUBLIC_API_URL environment variable
  • I-verify na gumagana ang blog posts mula sa Laravel API
  • Magdagdag ng custom domain (optional)
  • I-share ang iyong live portfolio URL!
