Lesson 0004: Git & GitHub for Freelancers

I-backup at i-track ang iyong code — parang "save point" sa video game!

Lesson Goal Matututunan mo ang Git basics: i-save ang versions ng code mo, mag-upload sa GitHub, at mag-collaborate sa ibang devs. Essential ito para sa freelancing — kailangan ng clients na may version control ka!

Ano ang Git at GitHub?

Git ay isang version control system — parang "save points" sa video game. Bawat save ay pwedeng balikan kung nagkamali ka. Pwede kang mag-experiment nang walang takot!

GitHub naman ay ang "cloud save" ng iyong Git repositories. Do mo ini-upload ang code mo para ma-access kahit saan, at para makita ng clients o employers.

Para sa freelancing: Ibig sabihin, kapag nasira mo ang code mo (at mangyayari yan!), pwede kang bumalik sa working version. At kapag may client kang gusto makita ang progress mo — bigay mo lang ang GitHub link mo!

Step 1: Install Git

1

Pumunta sa git-scm.com at i-download ang Git para sa Windows.

2

I-install ito — default settings lang, next-next lang. May kasama nang Git Bash na gamit natin.

3

I-open ang Git Bash at i-set ang name at email mo:

git config --global user.name "Juan Dela Cruz"
git config --global user.email "juan@example.com"
4

I-verify na naka-install na:

git --version
# Result: git version 2.x

Step 2: Create a GitHub Account

1

Pumunta sa github.com at mag-sign up — libre ang account!

2

Gumawa ng personal access token (ito na ang bagong password):

  • Settings → Developer settings → Personal access tokens → Tokens (classic)
  • Generate new token → i-check ang "repo" at "workflow"
  • I-save mo ang token! Hindi mo na ito makikita ulit.
Important Simula 2021, hindi na pwede ang password sa Git command line. Gumamit ng personal access token or GitHub CLI. I-treat ang token gaya ng password — huwag ipakita kahit kanino!

Git Basics — Ang mga Command na Dapat Mong Kabisaduhin

CommandGamit
git initGawing Git repository ang isang folder (isang beses lang)
git statusCheck kung ano ang nagbago sa files
git add .I-stage lahat ng changes para i-save
git add filename.htmlI-stage ang specific file lang
git commit -m "message"I-save ang staged changes (gumawa ng "save point")
git logTingnan ang history ng commits
git remote add origin URLI-connect sa GitHub repository
git push origin mainI-upload ang commits sa GitHub
git pull origin mainI-download ang latest changes mula sa GitHub

The Git Workflow

Eto ang main workflow na gagamitin mo araw-araw:

# 1. Mag-check ng status
git status

# 2. I-stage ang changes
git add .

# 3. I-commit (gumawa ng save point)
git commit -m "Added contact form with CSS styling"

# 4. I-upload sa GitHub
git push origin main
Good Commit Messages Sumulat ng malinaw na commit messages sa English. Hindi "update" o "fix" lang. Gawin mong descriptive:
git commit -m "Add responsive navigation bar with media queries"
git commit -m "Fix form validation bug on empty email field"

🛠️ Mini-Project: I-upload ang Portfolio Mo sa GitHub

Gawin Mo Ito!

I-upload ang portfolio.html mo sa GitHub — ito na ang unang repository mo!

1

Sa GitHub, i-click ang "New repository" (may "+" sign sa右上). Pangalanan ito na my-portfolio. Wag i-check ang "Initialize with README".

2

Sa Git Bash, pumunta sa folder ng portfolio mo at i-run:

cd /c/Users/Admin/Desktop/project/portfolio

git init
git add .
git commit -m "Initial commit: portfolio website with HTML, CSS, and JS"

git remote add origin https://github.com/YOUR_USERNAME/my-portfolio.git
git branch -M main
git push -u origin main

Palitan ang "YOUR_USERNAME" ng actual GitHub username mo.

3

I-refresh ang GitHub page mo — makikita mo na ang portfolio mo online! 🎉

4

I-share mo sa client! Ang URL na https://github.com/YOUR_USERNAME/my-portfolio ay pwedeng ipakita sa potential clients para makita nila ang code mo.

Create a .gitignore File

May mga file na hindi dapat i-upload sa GitHub — like node_modules, .env files, at operating system files. Gumawa ng .gitignore file sa root ng project mo:

# OS files
.DS_Store
Thumbs.db

# Dependencies
node_modules/
vendor/

# Environment
.env
.env.local

# IDE
.vscode/
.idea/

# Logs
*.log
Quick Start Pumunta sa gitignore.io at i-type ang "Windows", "Node", "VisualStudioCode" — mag-ge-generate ito ng .gitignore para sa'yo!

Undoing Changes — Kung Nagkamali Ka

# Undo changes sa isang file (ibalik sa last commit)
git checkout -- filename.html

# Unstage a file (bago pa ma-commit)
git reset HEAD filename.html

# Baliktarin ang last commit (keep changes)
git reset --soft HEAD~1

# Baliktarin ang last commit (remove changes completely)
git reset --hard HEAD~1

# Gumawa ng bagong commit para i-undo ang previous commit
git revert HEAD
Warning Ang git reset --hard ay permanently nagde-delete ng changes. Gamitin nang may pag-iingat! Kapag may pag-aalinlangan, git revert ang safer option.

Branching — Para sa mga Experiment

# Gumawa ng bagong branch
git branch feature-dark-mode

# Lumipat sa branch
git checkout feature-dark-mode
# or (shorter):
git switch feature-dark-mode

# Gumawa at lumipat in one command
git checkout -b feature-dark-mode

# I-merge ang branch sa main
git checkout main
git merge feature-dark-mode

# I-delete ang branch (optional)
git branch -d feature-dark-mode
Bakit magagamit ito sa freelancing? Kapag may client kang gusto magdagdag ng feature, gumawa ka ng bagong branch. Hindi mo sira ang main code. Kapag approve na, i-merge mo. Kung hindi approve, delete lang — walang nasira!

Collaboration Workflow (For Future Freelance Teams)

# Kunin ang latest code ng team
git pull origin main

# Gumawa ng branch para sa task mo
git checkout -b feature-new-page

# Gawin ang changes...
git add .
git commit -m "Add new about page"

# I-upload ang branch mo
git push -u origin feature-new-page

# Sa GitHub, gumawa ng Pull Request (PR)
# Hintaying ma-review at ma-approve
# Then merge!
Pull Request (PR) Ang PR ay para kang nagsasabi sa team, "Uy, may ginawa akong changes. Paki-check bago i-merge sa main." Ito ang standard workflow sa professional teams.

🎯 Lesson Summary

  • Git = version control (save points ng code mo)
  • GitHub = cloud storage para sa Git repositories
  • Main workflow: add → commit → push
  • Magandang commit messages: descriptive at nasa English
  • .gitignore — para i-exclude ang sensitive o unnecessary files
  • Branches — para magtrabaho sa bagong feature nang safe
  • Pull Request — para mag-pa-review ng code bago i-merge
  • Na-upload mo na ang portfolio mo sa GitHub! 🎉
