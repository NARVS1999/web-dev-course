Lesson 0007: Laravel Migrations, Eloquent ORM & CRUD

Ikonekta ang Laravel sa MySQL — gumawa, bumasa, mag-update, at magbura ng data!

Lesson Goal Matututunan mo kung paano ikonekta ang Laravel sa MySQL database, gumamit ng migrations para gumawa ng tables, Eloquent ORM para mag-interact sa database, at gumawa ng CRUD API (Create, Read, Update, Delete). Sa dulo, gagawa ka ng Blog API!

Ano ang Eloquent ORM?

Ang ORM (Object-Relational Mapping) ay isang technique na nagpapahintulot sa'yo na magtrabaho sa database gamit ang PHP objects at methods — hindi na kailangan magsulat ng SQL! 

// Without ORM — SQL
$users = DB::select('SELECT * FROM users WHERE age > ?', [18]);

// With Eloquent ORM — PHP objects!
$users = User::where('age', '>', 18)->get();
Analogy: Ang Eloquent ay parang translator — ikaw magsalita ng PHP, isasalin nito sa SQL para sa database. Hindi mo na kailangang mag-SQL nang manual!

Step 2: Migrations — Version Control para sa Database

Ang migrations ay parang Git para sa database schema. Sa halip na mag-execute ng SQL sa phpMyAdmin, magsusulat ka ng PHP code na naglalarawan ng tables mo. Pwedeng i-rollback, i-version, at i-share sa team!

1

Gumawa ng migration para sa posts table:

php artisan make:migration create_posts_table

Gagawa ito ng file sa database/migrations/xxxx_create_posts_table.php

2

I-edit ang migration file:

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('posts', function (Blueprint $table) {
            $table->id();
            $table->string('title', 200);
            $table->text('body');
            $table->foreignId('user_id')->constrained()->onDelete('cascade');
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('posts');
    }
};
Migration Methods Explained
  • up() — kung ano ang gagawin (gumawa ng table)
  • down() — kung paano i-undo (burahin ang table)
  • $table->id() — auto-increment primary key (named 'id')
  • $table->foreignId('user_id')->constrained() — foreign key na naka-link sa users table
  • $table->timestamps() — gumagawa ng created_at at updated_at columns
  • onDelete('cascade') — kapag na-delete ang user, mabubura rin ang posts niya
3

I-run ang migration:

php artisan migrate

Check sa phpMyAdmin — may posts table na! 🎉

4

Kung magkamali, pwedeng i-rollback:

php artisan migrate:rollback   
php artisan migrate:fresh
Naka-define na ang users table! Kapag gumawa ka ng bagong Laravel project, may kasama na itong create_users_table.php migration. Kasama na ang name, email, at password columns.

Step 3: Eloquent Models

Ang Model ay ang PHP class na kumakatawan sa isang database table.

1

Gumawa ng Post model (kasama na ang User model sa simula):

php artisan make:model Post

Gagawa ng file sa app/Models/Post.php

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Post extends Model
{
    // Specify kung aling columns ang pwedeng i-mass assign
    protected $fillable = [
        'title',
        'body',
        'user_id',
    ];

    // Define relationship: isang Post ay belongs sa isang User
    public function user()
    {
        return $this->belongsTo(User::class);
    }
}

I-edit din ang app/Models/User.php para i-define ang relationship:

<?php

namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;

class User extends Authenticatable
{
    protected $fillable = [
        'name',
        'email',
        'password',
    ];

    protected $hidden = [
        'password',
        'remember_token',
    ];

    // Define relationship: isang User ay may maraming Posts
    public function posts()
    {
        return $this->hasMany(Post::class);
    }
}

Step 4: Eloquent CRUD — Ang mga Basic Operations

CREATE — Gumawa ng Record

// Paraan 1: Create with array
$post = Post::create([
    'title' => 'My First Post',
    'body' => 'Hello, this is my first post using Eloquent!',
    'user_id' => 1,
]);

// Paraan 2: Create with instance
$post = new Post();
$post->title = 'Another Post';
$post->body = 'Body content here';
$post->user_id = 1;
$post->save();

dd($post); // check ang result

READ — Kumuha ng Records

// Lahat ng posts
$posts = Post::all();

// Hanapin by ID
$post = Post::find(1);

// May conditions
$posts = Post::where('user_id', 1)->get();
$post = Post::where('title', 'My First Post')->first();

// With relationships (para maiwasan ang N+1 problem)
$posts = Post::with('user')->get();

// Pagination
$posts = Post::paginate(10);

UPDATE — Baguhin ang Record

// Paraan 1: Find then update
$post = Post::find(1);
$post->title = 'Updated Title';
$post->save();

// Paraan 2: Direct update
Post::where('id', 1)->update([
    'title' => 'Updated Title Directly',
]);

DELETE — Burahin ang Record

// Paraan 1: Find then delete
$post = Post::find(1);
$post->delete();

// Paraan 2: Direct delete
Post::destroy(1);

// May condition
Post::where('user_id', 1)->delete();

🛠️ Mini-Project: Blog CRUD API

Gawin Mo Ito!

1

Gumawa ng PostController:

php artisan make:controller PostController --resource

Ang --resource flag ay gumagawa ng methods para sa CRUD: index, store, show, update, destroy.

2

I-edit ang app/Http/Controllers/PostController.php:

<?php

namespace App\Http\Controllers;

use App\Models\Post;
use Illuminate\Http\Request;

class PostController extends Controller
{
    // GET /api/posts — list all posts
    public function index()
    {
        $posts = Post::with('user')->get();
        return response()->json([
            'success' => true,
            'data' => $posts
        ]);
    }

    // POST /api/posts — create a post
    public function store(Request $request)
    {
        $post = Post::create([
            'title' => $request->title,
            'body' => $request->body,
            'user_id' => $request->user_id,
        ]);

        return response()->json([
            'success' => true,
            'data' => $post
        ], 201);
    }

    // GET /api/posts/{id} — show single post
    public function show($id)
    {
        $post = Post::with('user')->find($id);

        if (!$post) {
            return response()->json([
                'success' => false,
                'message' => 'Post not found'
            ], 404);
        }

        return response()->json([
            'success' => true,
            'data' => $post
        ]);
    }

    // PUT/PATCH /api/posts/{id} — update a post
    public function update(Request $request, $id)
    {
        $post = Post::find($id);

        if (!$post) {
            return response()->json([
                'success' => false,
                'message' => 'Post not found'
            ], 404);
        }

        $post->update([
            'title' => $request->title ?? $post->title,
            'body' => $request->body ?? $post->body,
        ]);

        return response()->json([
            'success' => true,
            'data' => $post
        ]);
    }

    // DELETE /api/posts/{id} — delete a post
    public function destroy($id)
    {
        $post = Post::find($id);

        if (!$post) {
            return response()->json([
                'success' => false,
                'message' => 'Post not found'
            ], 404);
        }

        $post->delete();

        return response()->json([
            'success' => true,
            'message' => 'Post deleted successfully'
        ]);
    }
}
3

I-edit ang routes/api.php:

<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\PostController;

// Resource route — automatic na gagawa ng lahat ng CRUD routes!
Route::apiResource('posts', PostController::class);

// Ito ang mabubuo:
// GET    /api/posts       → index()
// POST   /api/posts       → store()
// GET    /api/posts/{id}  → show()
// PUT    /api/posts/{id}  → update()
// DELETE /api/posts/{id}  → destroy()
