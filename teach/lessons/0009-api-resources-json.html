Lesson 0009: API Resources & JSON Responses

I-format at i-standardize ang iyong API responses gamit ang Laravel API Resources.

Lesson Goal Matutunan kung paano gumamit ng API Resources para mag-transform ng Eloquent models sa structured JSON responses, magdagdag ng pagination, at mag-handle ng errors nang maayos.

Part 1: Introduction to API Resources

Sa ngayon, ang ating API ay nagre-return ng raw Eloquent models bilang JSON. Problema: kasama ang lahat ng columns — pati na ang created_at at updated_at sa default format. Hindi ito maganda para sa isang propesyonal na API.

Ang API Resources ay isang transformation layer na nagbibigay-daan sa iyo na i-control kung ano ang lalabas sa iyong JSON response.

1

Create a Post Resource:

php artisan make:resource PostResource

May makikita kang bagong file sa app/Http/Resources/PostResource.php.

2

Edit PostResource:

<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class PostResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id'         => $this->id,
            'title'      => $this->title,
            'content'    => $this->content,
            'author'     => new UserResource($this->whenLoaded('user')),
            'comments'   => CommentResource::collection($this->whenLoaded('comments')),
            'created_at' => $this->created_at->diffForHumans(),
            'updated_at' => $this->updated_at->diffForHumans(),
        ];
    }
}

Ang $this sa loob ng resource ay tumutukoy sa model instance. Ang $this->whenLoaded() ay magre-return lang ng relationship data kung ito ay na-load (eager loaded).

3

Create UserResource:

php artisan make:resource UserResource
class UserResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id'   => $this->id,
            'name' => $this->name,
            'email' => $this->email,
        ];
    }
}
4

Create CommentResource:

php artisan make:resource CommentResource
class CommentResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id'     => $this->id,
            'author' => $this->author,
            'body'   => $this->body,
            'posted' => $this->created_at->diffForHumans(),
        ];
    }
}

Resource Collections

Para sa listahan ng posts, gumawa tayo ng PostCollection:

php artisan make:resource PostCollection
<?php

namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\ResourceCollection;

class PostCollection extends ResourceCollection
{
    public $collects = PostResource::class;

    public function toArray(Request $request): array
    {
        return [
            'data' => $this->collection,
            'meta' => [
                'total' => $this->total(),
                'count' => $this->count(),
                'per_page' => $this->perPage(),
                'current_page' => $this->currentPage(),
                'last_page' => $this->lastPage(),
            ],
        ];
    }
}
Resource vs Collection
  • PostResource — ginagamit para sa iisang Post model
  • PostCollection — ginagamit para sa listahan ng posts na may pagination
  • Alternative: Pwedeng direkta kang gumamit ng PostResource::collection($posts) para sa collections na walang meta

Part 2: Using Resources sa Routes

Ngayon, i-update natin ang routes/api.php para gamitin ang resources:

use App\Http\Resources\PostResource;
use App\Http\Resources\PostCollection;
use App\Models\Post;

// Public routes
Route::get('/posts', function () {
    $posts = Post::with(['user', 'comments'])->paginate(10);
    return new PostCollection($posts);
});

Route::get('/posts/{post}', function (Post $post) {
    $post->load(['user', 'comments']);
    return new PostResource($post);
});

// Protected routes (kailangan ng auth token)
Route::middleware('auth:sanctum')->group(function () {
    Route::post('/posts', function (Request $request) {
        $validated = $request->validate([
            'title'   => 'required|string|max:255',
            'content' => 'required|string',
        ]);

        $post = $request->user()->posts()->create($validated);

        return new PostResource($post);
    });

    Route::put('/posts/{post}', function (Request $request, Post $post) {
        $post->update($request->validate([
            'title'   => 'sometimes|string|max:255',
            'content' => 'sometimes|string',
        ]));

        return new PostResource($post);
    });

    Route::delete('/posts/{post}', function (Post $post) {
        $post->delete();
        return response()->json(['message' => 'Post deleted successfully']);
    });
});

Bakit maganda ang API Resources?
  • Consistency — lahat ng endpoints ay may pare-parehong JSON structure
  • Control — ikaw ang magde-decide kung anong data ang ilalabas
  • Transformations — pwedeng mag-format ng dates, i-hide ang sensitive data, at magdagdag ng computed fields
  • Conditional data — gamit ang whenLoaded(), when(), at iba pa

Part 3: Pagination at Response Structure

Ang paginate(10) ay automatic na nagdi-divide ng results sa pages. Ang PostCollection ay naglalagay ng meta object na may pagination info. Makukuha ng frontend ang mga ito para mag-display ng page navigation.

Ito ang standard na ginagamit ng karamihan sa modernong APIs (JSON:API inspired).

Mini-Project: Transform Blog API with Resources

Gawing propesyonal ang iyong Blog API:

  • Gumawa ng PostResource, UserResource, at CommentResource
  • Gumawa ng PostCollection na may pagination meta
  • I-update ang routes para gamitin ang mga resources
  • I-test ang output — dapat consistent ang JSON structure
  • Subukan ang pagination: GET /api/posts?page=1

Summary

  • Ang API Resources ay nagti-transform ng Eloquent models sa structured JSON
  • Gamitin ang php artisan make:resource para gumawa ng resource class
  • Ang whenLoaded() ay pumipigil sa N+1 query problems sa resources
  • Ang ResourceCollection ay ginagamit para sa mga listahan na may meta data
  • Ang paginate() method ay automatic na nagha-handle ng pagination
