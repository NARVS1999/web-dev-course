Lesson 0017: React Native API Integration

Ikonek ang iyong React Native app sa iyong Laravel API at mag-ipon ng data offline gamit ang AsyncStorage.

Lesson Goal Matutunan kung paano mag-fetch ng data mula sa Laravel API, mag-display ng lists gamit ang FlatList, at mag-save ng data locally gamit ang AsyncStorage.
Tech Stack Itinuturo ng lesson na ito ang Expo SDK 52 at React Native. Tiyaking ito ang bersyon na ginamit mo sa Lesson 0015. Kung magkaiba ang iyong version, maaaring mag-iba ang mga API.

Part 1: Setup — API Configuration

Gumawa tayo ng configuration file para sa API URL. Sa development, ito ay localhost:8000.

// config.js
const API_URL = 'http://localhost:8000/api';
export default API_URL;
Android Emulator Note Kung gumagamit ka ng Android emulator, localhost ay hindi tutugma sa iyong machine — gamitin ang 10.0.2.2 sa halip:
const API_URL = 'http://10.0.2.2:8000/api';
Sa physical phone naman, gamitin ang IP address ng iyong computer (hal. http://192.168.1.5:8000/api).

Part 2: Fetching Data with FlatList

Ang FlatList ay ang React Native na bersyon ng HTML lists — optimized para sa malalaking lists na may scroll at performance.

// screens/BlogListScreen.js
import { useState, useEffect } from 'react';
import {
  View, Text, FlatList, ActivityIndicator,
  TouchableOpacity, StyleSheet,
} from 'react-native';
import API_URL from '../config';

export default function BlogListScreen({ navigation }) {
  const [posts, setPosts] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(`${API_URL}/posts`)
      .then(res => res.json())
      .then(data => {
        setPosts(data.data || data);
        setLoading(false);
      })
      .catch(err => {
        console.error(err);
        setLoading(false);
      });
  }, []);

  if (loading) {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" color="#2563eb" />
      </View>
    );
  }

  return (
    <FlatList
      data={posts}
      keyExtractor={item => item.id.toString()}
      renderItem={({ item }) => (
        <TouchableOpacity
          style={styles.card}
          onPress={() => navigation.navigate('BlogDetail', { id: item.id })}
        >
          <Text style={styles.title}>{item.title}</Text>
          <Text style={styles.body} numberOfLines={2}>{item.body}</Text>
        </TouchableOpacity>
      )}
    />
  );
}

const styles = StyleSheet.create({
  center: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  card: {
    backgroundColor: '#fff',
    padding: 16,
    marginHorizontal: 16,
    marginVertical: 8,
    borderRadius: 8,
    elevation: 2,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 1 },
    shadowOpacity: 0.1,
    shadowRadius: 4,
  },
  title: { fontSize: 18, fontWeight: 'bold', marginBottom: 4 },
  body: { fontSize: 14, color: '#666' },
});
FlatList features May built-in na onEndReached (para sa infinite scroll/pagination), refreshControl (para sa pull-to-refresh), at ListHeaderComponent/ListEmptyComponent.

Part 4: AsyncStorage — Local Data

Ang AsyncStorage ay parang localStorage sa web — nagse-save ng data sa iyong phone. Gamitin ito para sa caching, user preferences, o offline data.

I-install muna:

npx expo install @react-native-async-storage/async-storage
import AsyncStorage from '@react-native-async-storage/async-storage';

// Save data
const saveData = async (key, value) => {
  try {
    await AsyncStorage.setItem(key, JSON.stringify(value));
  } catch (err) {
    console.error('Save error:', err);
  }
};

// Read data
const getData = async (key) => {
  try {
    const value = await AsyncStorage.getItem(key);
    return value ? JSON.parse(value) : null;
  } catch (err) {
    console.error('Read error:', err);
  }
};

// Remove data
const removeData = async (key) => {
  try {
    await AsyncStorage.removeItem(key);
  } catch (err) {
    console.error('Remove error:', err);
  }
};

Use Case: Caching API Data

useEffect(() => {
  const fetchPosts = async () => {
    // Try to get cached data first
    const cached = await getData('cached_posts');
    if (cached) {
      setPosts(cached);
      setLoading(false);
    }

    // Then fetch fresh data
    try {
      const res = await fetch(`${API_URL}/posts`);
      const data = await res.json();
      const postsData = data.data || data;
      setPosts(postsData);
      await saveData('cached_posts', postsData);
    } catch (err) {
      console.error('Fetch error:', err);
    } finally {
      setLoading(false);
    }
  };
  fetchPosts();
}, []);
Why cache? Para kahit walang internet, may data pa ring maipakita ang app. Ito ang offline-first approach — common sa mobile apps.

Part 5: Pull-to-Refresh

Gamit ang RefreshControl para mag-refresh ng data kapag nag-swipe pababa:

import { RefreshControl } from 'react-native';

const [refreshing, setRefreshing] = useState(false);

const onRefresh = async () => {
  setRefreshing(true);
  try {
    const res = await fetch(`${API_URL}/posts`);
    const data = await res.json();
    setPosts(data.data || data);
  } catch (err) {
    console.error(err);
  } finally {
    setRefreshing(false);
  }
};

// Sa FlatList:
<FlatList
  data={posts}
  keyExtractor={item => item.id.toString()}
  renderItem={renderItem}
  refreshControl={
    <RefreshControl refreshing={refreshing} onRefresh={onRefresh} />
  }
/>
