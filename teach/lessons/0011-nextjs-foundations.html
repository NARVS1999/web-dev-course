Lesson 0011: Next.js Foundations

Gawing moderno at mabilis ang iyong frontend gamit ang Next.js — ang React framework para sa production.

Lesson Goal Matutunan ang pundasyon ng Next.js: project setup, file-based routing, pages, layouts, at navigation — at ma-deploy ang isang multi-page portfolio site.

Part 1: Ano ang Next.js?

Ang Next.js ay isang React framework na nagbibigay ng:

  • Server-Side Rendering (SSR) — nagre-render ng React components sa server bago ipadala sa browser
  • Static Site Generation (SSG) — gumagawa ng HTML files sa build time
  • File-based Routing — ang filename mo na ang bahala sa routing (hindi na kailangan ng React Router)
  • API Routes — pwedeng magkaroon ng backend endpoints sa loob ng iyong frontend project
Bakit Next.js vs plain React? Sa plain React (Create React App), lahat ay client-side rendered — ibig sabihin, empty HTML ang natatanggap ng browser at JavaScript na ang gagawa ng lahat. Problema: mabagal ang initial load at hindi SEO-friendly. Solusyon: Next.js na may SSR/SSG.

Part 2: Project Setup

1

Install Node.js — kung wala ka pa, i-download sa nodejs.org (20.x LTS version).

2

Create a Next.js project (v15.x, Pages Router):

npx create-next-app@latest my-portfolio
cd my-portfolio

Sagutin ang prompts:

  • Would you like to use TypeScript? No (for now)
  • Would you like to use ESLint? Yes
  • Would you like to use Tailwind CSS? No
  • Would you like to use App Router? No (use Pages Router for now)
  • Would you like to customize the import alias? No

3

Run the development server:

npm run dev

Buksan ang http://localhost:3000 — makikita mo ang default Next.js page.

Project Structure

my-portfolio/
├── pages/           # Routes — bawat file ay isang page
│   ├── index.js     # → /
│   ├── about.js     # → /about
│   └── _app.js      # Root component (layout, global CSS)
├── public/          # Static files (images, fonts)
├── styles/          # CSS files
├── package.json
└── next.config.js
Pages Router vs App Router Simula sa Next.js 13, may dalawang routing systems. Ang Pages Router (gamit natin dito) ay ang traditional at mas madaling matutunan. Ang App Router ay ang bago na may React Server Components — mas powerful pero mas complex. Master muna natin ang Pages Router.

Part 3: Pages & File-Based Routing

Sa Next.js, ang routing ay file-based. Ibig sabihin, ang directory at filename mo na ang magde-determine ng URL:

FileURL
pages/index.js/
pages/about.js/about
pages/blog/index.js/blog
pages/blog/[slug].js/blog/my-post (dynamic)

Gumawa tayo ng mga page:

// pages/index.js — Home page
export default function Home() {
  return (
    <div>
      <h1>Welcome to My Portfolio</h1>
      <p>Hi! Ako si Juan — web developer.</p>
      <a href="/about">Learn more about me</a>
    </div>
  );
}
// pages/about.js — About page
export default function About() {
  return (
    <div>
      <h1>About Me</h1>
      <p>I build web apps with Laravel & Next.js.</p>
      <a href="/">Back to Home</a>
    </div>
  );
}

Navigation with Link

Huwag gumamit ng regular na <a> tag para sa internal navigation — gumamit ng Link component para sa client-side navigation (walang page reload):

import Link from 'next/link';

export default function Home() {
  return (
    <div>
      <h1>Welcome to My Portfolio</h1>
      <p>Hi! Ako si Juan — web developer.</p>
      <Link href="/about">Learn more about me</Link>
    </div>
  );
}
Important Ang Link ay client-side navigation — hindi nagre-reload ang page, mas mabilis ang transitions. Pero ang <a> naman ay magti-trigger ng full page reload. Gamitin ang Link para sa internal links at <a> para sa external links.

Part 4: Shared Layout

Sa halip na ulit-ulitin ang header at footer sa bawat page, gumawa tayo ng layout component:

// components/Layout.js
import Link from 'next/link';

export default function Layout({ children }) {
  return (
    <div>
      <nav>
        <Link href="/">Home</Link>
        <Link href="/about">About</Link>
        <Link href="/blog">Blog</Link>
      </nav>
      <main>{children}</main>
      <footer>&copy; 2026 My Portfolio</footer>
    </div>
  );
}

At gamitin ito sa pages/_app.js para ma-apply sa lahat ng pages:

// pages/_app.js
import Layout from '../components/Layout';
import '../styles/globals.css';

export default function App({ Component, pageProps }) {
  return (
    <Layout>
      <Component {...pageProps} />
    </Layout>
  );
}
How _app.js works Ang _app.js ay ang root component ng Next.js — lahat ng pages ay dumadaan dito. Dito mo nilalagay ang global styles, layout, at providers (context, auth, atbp.).

Mini-Project: Multi-Page Portfolio

Gawin ang iyong personal portfolio website gamit ang Next.js:

  • Gumawa ng Next.js project: npx create-next-app@latest my-portfolio
  • Gumawa ng Home page (/) na may introduction
  • Gumawa ng About page (/about) na may skills at experience
  • Gumawa ng Blog page (/blog) na may listahan ng articles
  • Gumawa ng Layout component na may navigation
  • I-apply ang layout sa _app.js
  • Gamitin ang Link component sa navigation

Summary

  • Ang Next.js ay isang React framework na may SSR, SSG, at file-based routing
  • Ang create-next-app ang pinakamabilis na paraan para magsimula
  • Ang file-based routing ay gumagamit ng filesystem para sa URL structure
  • Ang Link component ay para sa client-side navigation
  • Ang _app.js ay para sa global layouts at styles
