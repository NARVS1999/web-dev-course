Lesson 0006: Laravel Setup, Routing & Controllers

Gawa tayo ng backend API gamit ang Laravel — ang puso ng iyong web app!

Lesson Goal I-setup ang Laravel sa local machine, intindihin ang routing system, controllers, at Blade templates. Sa dulo, gagawa ka ng "Hello World" API at isang simple web page gamit ang local server!

Ano ang Laravel?

Ang Laravel ay ang pinakasikat na PHP framework para sa web development. Ginagamit ito ng malalaking companies at freelancers dahil:

  • Mabilis mag-setup at mag-develop
  • Secure — built-in na ang protection sa common attacks
  • ORM (Eloquent) — napakadaling magtrabaho sa database
  • Massive ecosystem — maraming tools at libraries
  • Perfect for freelancing — maraming clients ang gumagamit ng Laravel
Analogy: Kung ang PHP ay parang martilyo at pako (pwede mong gamitin pero ikaw gagawa ng lahat), ang Laravel naman ay parang power tools — may built-in na lahat ng kailangan mo para makagawa ng professional web app nang mabilis.

Step 1: Install Composer

Ang Composer ay ang package manager ng PHP — parang npm para sa JavaScript.

1

Pumunta sa getcomposer.org at i-download ang Composer for Windows.

2

I-install ito. Hanapin nito ang PHP path mula sa XAMPP mo (C:\xampp\php\php.exe).

3

I-verify sa command prompt:

composer --version
XAMPP + Composer Kung hindi ma-detect ng Composer installer ang PHP, i-manual set ang path: i-add ang C:\xampp\php sa System Environment Variables > Path.

Step 2: Create Laravel Project

1

I-open ang Command Prompt (or Git Bash) at pumunta sa folder kung saan mo gusto ilagay ang project:

cd C:\Users\Admin\Desktop\project
Requirement: PHP 8.2+ Siguraduhing ang iyong XAMPP ay may PHP 8.2 o mas bago. I-check sa command prompt: php --version.
2

Gumawa ng bagong Laravel project (v11.x recommended):

composer create-project laravel/laravel "^11.0" backend-api

cd backend-api

Matagal ito — nagda-download ng dependencies. Mga 2-5 minutes.

3

I-start ang development server:

php artisan serve

Pupunta ito sa http://localhost:8000 — makikita mo ang Laravel welcome page! 🎉

Artisan Ang php artisan ay ang command-line tool ng Laravel. Marami kang magagawa dito: gumawa ng controllers, migrations, models, at marami pang iba. Kabisaduhin mo ito!

Laravel Directory Structure (Mga Kailangan Mong Malaman)

backend-api/
├── app/
│   ├── Http/
│   │   ├── Controllers/    ← Dito ang controllers mo
│   │   └── Middleware/      ← Request filters
│   └── Models/             ← Eloquent models (database tables)
├── bootstrap/              ← App bootstrapping (wag galawin)
├── config/                 ← Configuration files
├── database/
│   └── migrations/         ← Database migrations
├── public/                 ← Document root (index.php)
├── resources/
│   └── views/              ← Blade templates (HTML)
├── routes/
│   ├── web.php             ← Web routes (with sessions)
│   └── api.php             ← API routes (stateless)
├── storage/                ← Logs, cache, uploaded files
├── .env                    ← Environment variables (DB credentials!)
├── artisan                 ← Artisan CLI
└── composer.json           ← Dependencies

Routing — Ang Entry Point

Ang routes ang nagdi-determine kung anong code ang tatakbo kapag may nag-access ng specific URL.

routes/web.php — Para sa Web Pages

<?php

use Illuminate\Support\Facades\Route;

// Basic route
Route::get('/', function () {
    return view('welcome');
});

// Route na may parameter
Route::get('/hello/{name}', function ($name) {
    return "Hello, " . $name . "!";
});

// Route na may optional parameter
Route::get('/user/{id?}', function ($id = null) {
    if ($id) {
        return "User ID: " . $id;
    }
    return "No user specified.";
});

routes/api.php — Para sa REST API

<?php

use Illuminate\Support\Facades\Route;

// API routes — auto-prefixed with /api
Route::get('/hello', function () {
    return response()->json([
        'message' => 'Hello, World!',
        'status' => 'success'
    ]);
});

Route::get('/greet/{name}', function ($name) {
    return response()->json([
        'message' => "Hello, {$name}!",
        'time' => now()->toDateTimeString()
    ]);
});
Web vs API Routes Ang web.php ay para sa mga pages na may session at CSRF protection (like Blade views). Ang api.php ay para sa REST APIs na stateless — ito ang gagamitin natin for our freelance projects para pwedeng kumonekta ang Next.js at React Native!

Controllers — Pag-Organize ng Code

Sa halip na ilagay lahat ng logic sa routes, gumamit ng controllers para i-organize ang code.

1

Gumawa ng controller gamit ang Artisan:

php artisan make:controller UserController

Gagawa ito ng file sa app/Http/Controllers/UserController.php

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    public function index()
    {
        return response()->json([
            'users' => [
                ['id' => 1, 'name' => 'Juan'],
                ['id' => 2, 'name' => 'Maria'],
                ['id' => 3, 'name' => 'Pedro'],
            ]
        ]);
    }

    public function show($id)
    {
        return response()->json([
            'id' => $id,
            'name' => 'User ' . $id,
        ]);
    }
}

Kumonekta ang Route sa Controller

// routes/api.php
use App\Http\Controllers\UserController;

Route::get('/users', [UserController::class, 'index']);
Route::get('/users/{id}', [UserController::class, 'show']);

Blade Templates — Mag-render ng HTML

Ang Blade ay ang templating engine ng Laravel. Ginagamit ito para mag-render ng HTML na may dynamic data.

<!-- resources/views/hello.blade.php -->
<!DOCTYPE html>
<html>
<head>
    <title>Blade Hello</title>
</head>
<body>
    <h1>Hello, {{ $name }}!</h1>
    <p>Current time: {{ date('Y-m-d H:i:s') }}</p>
</body>
</html>
<?php
// routes/web.php
Route::get('/blade-hello/{name}', function ($name) {
    return view('hello', ['name' => $name]);
});

🛠️ Mini-Project: Gawa Tayo ng "Hello World" API

Gawin Mo Ito!

1

Gumawa ng bagong controller:

php artisan make:controller GreetingController
2

I-edit ang app/Http/Controllers/GreetingController.php:

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class GreetingController extends Controller
{
    public function hello()
    {
        return response()->json([
            'message' => 'Hello, World!',
            'from' => 'Laravel API',
            'timestamp' => now()->toDateTimeString()
        ]);
    }

    public function greet($name)
    {
        return response()->json([
            'message' => "Hello, {$name}! Welcome to Laravel.",
            'timestamp' => now()->toDateTimeString()
        ]);
    }
}
3

I-edit ang routes/api.php:

<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\GreetingController;

Route::get('/hello', [GreetingController::class, 'hello']);
Route::get('/greet/{name}', [GreetingController::class, 'greet']);
4

I-test sa browser o Postman:

http://localhost:8000/api/hello
→ {"message":"Hello, World!","from":"Laravel API","timestamp":"2026-07-04 12:00:00"}

http://localhost:8000/api/greet/Juan
→ {"message":"Hello, Juan! Welcome to Laravel.","timestamp":"2026-07-04 12:00:00"}
