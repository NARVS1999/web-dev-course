Lesson 0008: Laravel Relationships & Authentication

Ikonekta ang mga modelo gamit ang Eloquent relationships at protektahan ang iyong API gamit ang Sanctum auth.

Lesson Goal Sa lesson na ito, matututunan mo ang one-to-many relationships sa Eloquent at kung paano magdagdag ng authentication sa iyong Laravel API gamit ang Sanctum.
Prerequisite Kailangan mo munang matapos ang Lesson 0007 — dapat may working Blog API ka na may Post model at database migration.

Part 1: Eloquent Relationships

Sa Lesson 0007, gumawa ka ng Post model na may table sa database. Ngayon magdadagdag tayo ng Comment model para sa comments ng bawat post — ito ay one-to-many relationship.

One-to-Many Relationship Pattern

Think of it this way Isang Post ay pwedeng magkaroon ng maraming Comments → Post hasMany Comment.
Ang bawat Comment ay belong sa iisang Post → Comment belongsTo Post.

Gawin natin ito step-by-step.

1

Create Comment model with migration:

php artisan make:model Comment -m

Ang -m flag ay gumagawa ng migration file kasabay ng model.

2

Edit the migration file database/migrations/xxxx_create_comments_table.php:

public function up(): void
{
    Schema::create('comments', function (Blueprint $table) {
        $table->id();
        $table->foreignId('post_id')->constrained()->cascadeOnDelete();
        $table->string('author');
        $table->text('body');
        $table->timestamps();
    });
}

Ang foreignId('post_id')->constrained() ay gumagawa ng foreign key na naka-link sa id column ng posts table. Ang cascadeOnDelete ay automatic na magde-delete ng comments kapag ang post ay na-delete.

3

Run the migration:

php artisan migrate

Defining Relationships sa Models

Buksan ang app/Models/Post.php at idagdag ang relationship:

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Post extends Model
{
    protected $fillable = ['title', 'content'];

    public function comments()
    {
        return $this->hasMany(Comment::class);
    }
}

Ngayon, buksan ang app/Models/Comment.php at idagdag ang inverse relationship:

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Comment extends Model
{
    protected $fillable = ['post_id', 'author', 'body'];

    public function post()
    {
        return $this->belongsTo(Post::class);
    }
}

Gamitin ang Relationships

Sa iyong routes/api.php, magdagdag ng nested route para sa comments:

use App\Models\Post;
use App\Models\Comment;

// GET /api/posts — list all posts (with comments)
Route::get('/posts', function () {
    return Post::with('comments')->get();
});

// GET /api/posts/{post} — single post with comments
Route::get('/posts/{post}', function (Post $post) {
    return $post->load('comments');
});

// POST /api/posts/{post}/comments — add a comment
Route::post('/posts/{post}/comments', function (Post $post, Request $request) {
    $validated = $request->validate([
        'author' => 'required|string|max:255',
        'body'   => 'required|string',
    ]);

    $comment = $post->comments()->create($validated);

    return response()->json($comment, 201);
});

// DELETE /api/comments/{comment} — delete a comment
Route::delete('/comments/{comment}', function (Comment $comment) {
    $comment->delete();
    return response()->noContent();
});
Eager Loading Ang with('comments') ay naglo-load ng comments sa iisang query lang (eager loading) — hindi na kailangan ng maraming database queries. Ito ay best practice para sa performance.

Part 2: Laravel Sanctum Authentication

Ngayong may comments na ang blog API natin, kailangan nating protektahan ito. Hindi dapat makapag-delete ng comments ang kahit sino. Dito papasok ang Laravel Sanctum — isang lightweight authentication package para sa APIs.

1

Install Sanctum:

composer require laravel/sanctum

php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"

php artisan migrate

Magkakaroon ito ng personal_access_tokens table sa database.

2

Add Sanctum trait sa User model (app/Models/User.php):

<?php

namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;
use Laravel\Sanctum\HasApiTokens;

class User extends Authenticatable
{
    use HasApiTokens;
    // ...
}

Auth Routes: Register at Login

Sa routes/api.php, magdagdag ng public auth routes:

use App\Models\User;
use Illuminate\Support\Facades\Hash;
use Illuminate\Validation\ValidationException;

// POST /api/register
Route::post('/register', function (Request $request) {
    $validated = $request->validate([
        'name'     => 'required|string|max:255',
        'email'    => 'required|email|unique:users',
        'password' => 'required|string|min:8|confirmed',
    ]);

    $user = User::create([
        'name'     => $validated['name'],
        'email'    => $validated['email'],
        'password' => Hash::make($validated['password']),
    ]);

    $token = $user->createToken('api-token')->plainTextToken;

    return response()->json(['user' => $user, 'token' => $token], 201);
});

// POST /api/login
Route::post('/login', function (Request $request) {
    $validated = $request->validate([
        'email'    => 'required|email',
        'password' => 'required|string',
    ]);

    $user = User::where('email', $validated['email'])->first();

    if (! $user || ! Hash::check($validated['password'], $user->password)) {
        throw ValidationException::withMessages([
            'email' => ['The provided credentials are incorrect.'],
        ]);
    }

    $token = $user->createToken('api-token')->plainTextToken;

    return response()->json(['user' => $user, 'token' => $token]);
});

Protecting Routes

Ngayon, protektahan natin ang routes na nangangailangan ng authentication. Gamitin ang auth:sanctum middleware:

// Protected routes — nangangailangan ng valid token
Route::middleware('auth:sanctum')->group(function () {

    // GET /api/user — get current user info
    Route::get('/user', function (Request $request) {
        return $request->user();
    });

    // POST /api/posts — create post (kailangan naka-login)
    Route::post('/posts', function (Request $request) {
        $validated = $request->validate([
            'title'   => 'required|string|max:255',
            'content' => 'required|string',
        ]);

        $post = $request->user()->posts()->create($validated);

        return response()->json($post, 201);
    });

    // DELETE /api/posts/{post} — delete post (owner only)
    Route::delete('/posts/{post}', function (Post $post) {
        if ($request->user()->id !== $post->user_id) {
            return response()->json(['message' => 'Forbidden'], 403);
        }
        $post->delete();
        return response()->noContent();
    });

    // DELETE /api/comments/{comment}
    Route::delete('/comments/{comment}', function (Comment $comment) {
        $comment->delete();
        return response()->noContent();
    });
});
How to use the token Kapag nag-login o nag-register ang user, makakakuha sila ng token. Isama ang token na ito sa lahat ng protected requests bilang Bearer token sa Authorization header:

Authorization: Bearer 1|a1b2c3d4e5f6...

I-update ang Post Model

Kailangan din nating i-update ang Post model para magkaroon ng user relationship:

class Post extends Model
{
    protected $fillable = ['user_id', 'title', 'content'];

    public function user()
    {
        return $this->belongsTo(User::class);
    }

    public function comments()
    {
        return $this->hasMany(Comment::class);
    }
}

At magdagdag ng user_id column sa posts table sa pamamagitan ng bagong migration:

php artisan make:migration add_user_id_to_posts_table
public function up(): void
{
    Schema::table('posts', function (Blueprint $table) {
        $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    });
}
php artisan migrate

Mini-Project: Comments + Auth Integration

Gawin nating kumpleto ang blog API:

  • Magdagdag ng Comment model na may post_id, author, at body
  • I-define ang hasMany / belongsTo relationships
  • I-install ang Sanctum at mag-setup ng register/login endpoints
  • I-protect ang create/delete routes gamit ang auth:sanctum
  • I-test ang iyong API gamit ang curl o Postman

Test with curl:

# Register
curl -X POST http://localhost:8000/api/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Juan","email":"juan@example.com","password":"password123","password_confirmation":"password123"}'

# Login (copy the token from response)
curl -X POST http://localhost:8000/api/login \
  -H "Content-Type: application/json" \
  -d '{"email":"juan@example.com","password":"password123"}'

# Create a post (replace TOKEN with actual token)
curl -X POST http://localhost:8000/api/posts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{"title":"My First Post","content":"Hello World!"}'

# Add a comment (no auth needed)
curl -X POST http://localhost:8000/api/posts/1/comments \
  -H "Content-Type: application/json" \
  -d '{"author":"Maria","body":"Nice post!"}'

Summary

  • Ang hasMany() at belongsTo() ay ginagamit para sa one-to-many relationships
  • Ang foreignId()->constrained() ay gumagawa ng foreign key sa migration
  • Ang with() ay para sa eager loading ng relationships
  • Ang Laravel Sanctum ay nagbibigay ng API token authentication
  • Gamitin ang auth:sanctum middleware para protektahan ang routes
  • Ang token ay isinasama bilang Authorization: Bearer header
