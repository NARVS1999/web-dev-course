Lesson 0010: File Uploads & Storage

Mag-upload, mag-validate, at mag-serve ng files gamit ang Laravel Storage.

Lesson Goal Matutunan kung paano mag-handle ng file uploads sa Laravel — kasama ang validation, storage (local), at pag-serve ng uploaded files sa pamamagitan ng API.

Part 1: Laravel Storage

Ang Laravel Storage ay isang abstraction layer para sa file storage. Pwedeng local (sa iyong server) o cloud (Amazon S3, DigitalOcean Spaces, atbp.). Sa ngayon, local storage muna ang gagamitin natin.

Ang default na local storage ay nagse-save ng files sa storage/app/ directory. Pero para ma-access ang mga ito sa web, kailangan nating gumawa ng symbolic link mula sa public/storage papunta sa storage/app/public.

1

Create the storage link:

php artisan storage:link

Magagawa nito ang isang symbolic link: public/storage → storage/app/public. Ngayon, ang mga files na nasa storage/app/public ay maa-access sa browser sa pamamagitan ng http://localhost:8000/storage/filename.jpg.

Part 2: Profile Picture Upload (Avatar)

Magdagdag tayo ng avatar upload feature sa ating User model.

1

Create a migration para sa avatar column:

php artisan make:migration add_avatar_to_users_table
public function up(): void
{
    Schema::table('users', function (Blueprint $table) {
        $table->string('avatar')->nullable()->after('email');
    });
}
php artisan migrate
2

Add avatar sa $fillable ng User model:

protected $fillable = ['name', 'email', 'password', 'avatar'];
3

Create the upload route sa routes/api.php:

use Illuminate\Http\Request;
use Illuminate\Support\Facades\Storage;

Route::middleware('auth:sanctum')->group(function () {

    // POST /api/avatar — upload profile picture
    Route::post('/avatar', function (Request $request) {
        $request->validate([
            'avatar' => 'required|image|mimes:jpeg,png,jpg,gif,webp|max:2048',
        ]);

        $path = $request->file('avatar')->store('avatars', 'public');

        $user = $request->user();
        $user->avatar = $path;
        $user->save();

        $url = Storage::url($path);

        return response()->json([
            'message' => 'Avatar uploaded successfully',
            'avatar'  => $url,
        ]);
    });

    // GET /api/user — get user profile with avatar URL
    Route::get('/user', function (Request $request) {
        $user = $request->user();
        return response()->json([
            'id'     => $user->id,
            'name'   => $user->name,
            'email'  => $user->email,
            'avatar' => $user->avatar ? Storage::url($user->avatar) : null,
        ]);
    });
});
Validation Rules Explained
  • required — kailangang may file
  • image — dapat ito ay isang image file (JPEG, PNG, GIF, BMP, o WebP)
  • mimes:jpeg,png,jpg,gif,webp — specific file types lang ang allowed
  • max:2048 — maximum file size sa kilobytes (2048KB = 2MB)

Testing the Upload

Gamit ang curl, i-test ang avatar upload:

# Login first and get the token
TOKEN="your-api-token-here"

# Upload avatar
curl -X POST http://localhost:8000/api/avatar \
  -H "Authorization: Bearer $TOKEN" \
  -F "avatar=@/path/to/profile.jpg"

# Expected response:
# {
#   "message": "Avatar uploaded successfully",
#   "avatar": "http://localhost:8000/storage/avatars/abc123.jpg"
# }

# Get user profile (with avatar URL)
curl http://localhost:8000/api/user \
  -H "Authorization: Bearer $TOKEN"

Part 3: Storage Methods at Best Practices

Common Storage Methods

Method Description
Storage::put('path', $contents) Mag-save ng file gamit ang string contents
$file->store('path') Mag-store ng uploaded file sa given path
$file->storeAs('path', 'name.jpg') Mag-store na may specific filename
Storage::url('path') Kunin ang public URL ng file
Storage::delete('path') Delete ng file
Storage::exists('path') Check kung may file
Security Reminder
  • Laging mag-validate ng uploaded files — huwag basta-basta magtiwala sa filename o extension
  • Gamitin ang image rule para sa images — hindi lang extension ang chine-check nito, pati ang actual file content
  • Mag-set ng maximum file size para maiwasan ang server overload
  • Huwag gamitin ang original filename mula sa user — gamitin ang store() na nag-ge-generate ng unique filename

Deleting Old Avatars

Kapag nag-upload ng bagong avatar, dapat idelete ang luma para hindi kumain ng disk space:

use Illuminate\Support\Facades\Storage;

Route::middleware('auth:sanctum')->post('/avatar', function (Request $request) {
    $request->validate([
        'avatar' => 'required|image|mimes:jpeg,png,jpg,gif,webp|max:2048',
    ]);

    $user = $request->user();

    // Delete old avatar kung mayroon
    if ($user->avatar && Storage::disk('public')->exists($user->avatar)) {
        Storage::disk('public')->delete($user->avatar);
    }

    // Store new avatar
    $path = $request->file('avatar')->store('avatars', 'public');
    $user->avatar = $path;
    $user->save();

    return response()->json([
        'message' => 'Avatar updated successfully',
        'avatar'  => Storage::url($path),
    ]);
});
