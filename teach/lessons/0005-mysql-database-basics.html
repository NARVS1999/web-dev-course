Lesson 0005: MySQL & Database Basics

Ang pundasyon ng data — kung paano nag-iimbak at kumukuha ng impormasyon ang iyong apps!

Lesson Goal Matututunan mo ang database fundamentals: tables, SQL queries, keys, at relationships. I-install mo ang XAMPP, gagawa ng database gamit ang phpMyAdmin, at magsusulat ng SQL queries — lahat sa local machine mo nang libre!

Ano ang Database?

Ang database ay isang organisadong koleksyon ng data. Isipin mo na lang na parang Excel spreadsheet — may rows at columns. Pero mas powerful ang databases dahil:

  • Milyon-milyong records kaya mong i-handle
  • Mabilis mag-search at mag-filter
  • Pwedeng mag-relate ang iba't ibang tables sa isa't isa
  • Secure at may user permissions

Ang MySQL ay isa sa pinakasikat na database management system. Libre ito at gamit na gamit sa web development — lalo na sa Laravel.

Analogy: Ang database ay parang library. Ang tables ay ang mga bookshelves (isa para sa books, isa para sa authors, isa para sa borrowers). Ang rows ay ang mga indibidwal na libro. Ang columns ay ang impormasyon tungkol sa bawat libro (title, author, year, ISBN).

Database Concepts

Tables, Rows, at Columns

Table: users
+----+--------+---------------------+-----------+
| id | name   | email               | age       |
+----+--------+---------------------+-----------+
| 1  | Juan   | juan@example.com    | 25        |
| 2  | Maria  | maria@example.com   | 30        |
| 3  | Pedro  | pedro@example.com   | 22        |
+----+--------+---------------------+-----------+
  ↑      ↑           ↑                  ↑
  id     name        email              age
(column)(column)    (column)          (column)
  • Table — ang lalagyan ng data (parang Excel sheet)
  • Row — isang record (isang user)
  • Column — isang attribute ng record (name, email)

Primary Key

Ang Primary Key ay isang column (or combination) na unique na nag-iidentify sa bawat row. Usually id na auto-increment. 

CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,  ← eto ang primary key
  name VARCHAR(100),
  email VARCHAR(255)
);
Auto-Increment Ang AUTO_INCREMENT ay automatic nagbibigay ng number: 1, 2, 3, 4... Hindi mo na kailangang i-set manually.

Foreign Key

Ang Foreign Key ay isang column na nagli-link ng isang table sa primary key ng ibang table. Dito nagkakaroon ng relationships ang databases. 

Table: posts
+----+----------------+---------+
| id | title          | user_id |  ← foreign key (links to users.id)
+----+----------------+---------+
| 1  | My First Post  | 1       |  ← si Juan ang may-ari
| 2  | Hello World    | 2       |  ← si Maria ang may-ari
+----+----------------+---------+

Table: users
+----+-------+
| id | name  |
+----+-------+
| 1  | Juan  |
| 2  | Maria |
+----+-------+

SQL — Ang Language ng Databases

Ang SQL (Structured Query Language) ang ginagamit para makipag-usap sa database. Parang "English" para sa databases.

CREATE — Gumawa ng Table

CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) NOT NULL UNIQUE,
  age INT DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

Key constraints:

  • NOT NULL — required, hindi pwedeng blanko
  • UNIQUE — bawal mag-duplicate (like email)
  • DEFAULT value — automatic value kung hindi naman provided

INSERT — Maglagay ng Data

INSERT INTO users (name, email, age)
VALUES ('Juan', 'juan@example.com', 25);

INSERT INTO users (name, email, age)
VALUES ('Maria', 'maria@example.com', 30),
       ('Pedro', 'pedro@example.com', 22);

SELECT — Kumuha ng Data

-- Kunin lahat
SELECT * FROM users;

-- Specific columns lang
SELECT name, email FROM users;

-- May condition
SELECT * FROM users WHERE age >= 25;

-- May sorting
SELECT * FROM users ORDER BY name ASC;
SELECT * FROM users ORDER BY age DESC;

-- Limitahan ang results
SELECT * FROM users LIMIT 10;

-- Count
SELECT COUNT(*) FROM users;
SELECT COUNT(*) FROM users WHERE age > 20;

UPDATE — Baguhin ang Data

UPDATE users
SET age = 26
WHERE id = 1;

-- Mag-update ng multiple columns
UPDATE users
SET name = 'Juan Dela Cruz', age = 26
WHERE id = 1;
Warning! Huwag kakalimutan ang WHERE sa UPDATE at DELETE! Kapag nakalimutan mo, lahat ng rows ang maaapektuhan!

DELETE — Burahin ang Data

DELETE FROM users WHERE id = 1;

-- Burahin lahat (ingat!)
DELETE FROM users;

WHERE — Mga Conditions

SELECT * FROM users WHERE age = 25;
SELECT * FROM users WHERE age > 25;
SELECT * FROM users WHERE age >= 25 AND age <= 35;
SELECT * FROM users WHERE name LIKE '%an%';    -- may "an" sa name
SELECT * FROM users WHERE email IS NOT NULL;
SELECT * FROM users WHERE id IN (1, 3, 5);

Joins — Pagsasama ng Tables

Ito ang pinakamahalaga sa relational databases — ang pagsama ng data mula sa dalawa o higit pang tables.

INNER JOIN

Kunin lang ang may match sa parehong tables:

SELECT users.name, posts.title
FROM users
INNER JOIN posts ON users.id = posts.user_id;

+-------+----------------+
| name  | title          |
+-------+----------------+
| Juan  | My First Post  |
| Maria | Hello World    |
+-------+----------------+
-- Si Pedro wala pang post, hindi kasama

LEFT JOIN

Kunin lahat mula sa left table, kahit walang match sa right:

SELECT users.name, posts.title
FROM users
LEFT JOIN posts ON users.id = posts.user_id;

+-------+----------------+
| name  | title          |
+-------+----------------+
| Juan  | My First Post  |
| Maria | Hello World    |
| Pedro | NULL           | ← walang post si Pedro
+-------+----------------+

🛠️ Mini-Project: Gawin ang Database ng Blog

Gawin Mo Ito sa phpMyAdmin!

1

I-open ang http://localhost/phpmyadmin

2

I-click ang New sa left sidebar at gumawa ng database na blog_db

3

I-click ang SQL tab at i-paste ito:

CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) NOT NULL UNIQUE,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
  id INT AUTO_INCREMENT PRIMARY KEY,
  title VARCHAR(200) NOT NULL,
  body TEXT NOT NULL,
  user_id INT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

INSERT INTO users (name, email) VALUES
('Juan Dela Cruz', 'juan@example.com'),
('Maria Santos', 'maria@example.com');

INSERT INTO posts (title, body, user_id) VALUES
('My First Blog Post', 'Hello everyone! This is my first post.', 1),
('Learning SQL', 'SQL is fun and powerful!', 1),
('Web Dev Tips', 'Here are some tips for beginners...', 2);
4

I-click ang Go — makikita mong may tables na!

5

I-try ang mga SQL queries na ito sa SQL tab:

-- Lahat ng users at kanilang posts
SELECT users.name, posts.title, posts.body
FROM users
INNER JOIN posts ON users.id = posts.user_id;

-- Ilang posts ang bawat user?
SELECT users.name, COUNT(posts.id) AS post_count
FROM users
LEFT JOIN posts ON users.id = posts.user_id
GROUP BY users.id;
