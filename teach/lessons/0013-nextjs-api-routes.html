Lesson 0013: Next.js API Routes

Gumawa ng backend endpoints sa loob mismo ng iyong Next.js app — perfect para sa mga form submissions at authentication.

Lesson Goal Matutunan kung paano gumawa ng API routes sa Next.js, mag-handle ng form submissions, gumamit ng middleware, at mag-setup ng environment variables para sa secure na API calls.

Part 1: Introduction to API Routes

Ang API Routes ay nagbibigay-daan sa iyo na gumawa ng serverless API endpoints sa loob ng iyong Next.js project. Ilagay ang mga ito sa pages/api/ directory — same file-based routing pattern gamit.

FileURL
pages/api/hello.js/api/hello
pages/api/contact.js/api/contact
pages/api/posts/[id].js/api/posts/1
Note Ang API Routes ay hindi kapalit ng Laravel backend. Gamitin ang mga ito para sa:
  • Form submissions (contact form)
  • Authentication (NextAuth.js)
  • Proxy — mag-fetch mula sa external API (para maiwasan ang CORS issues)
  • Server-side operations na kailangan ng secret keys

Part 2: Creating API Routes

1

Create a basic API route:

// pages/api/hello.js
export default function handler(req, res) {
  res.status(200).json({ message: 'Hello from Next.js!' });
}

I-test: http://localhost:3000/api/hello → {"message":"Hello from Next.js!"}

2

Handle different HTTP methods:

// pages/api/contact.js
export default function handler(req, res) {
  if (req.method === 'POST') {
    // Process the form submission
    const { name, email, message } = req.body;

    // Validate
    if (!name || !email || !message) {
      return res.status(400).json({ error: 'All fields are required.' });
    }

    // Send email or save to database (dummy example)
    console.log('Contact form:', { name, email, message });

    return res.status(200).json({ message: 'Message sent successfully!' });
  }

  // Handle other methods
  res.setHeader('Allow', ['POST']);
  res.status(405).json({ error: `Method ${req.method} not allowed` });
}
3

Connect form to API route:

// components/ContactForm.js
import { useState } from 'react';

export default function ContactForm() {
  const [status, setStatus] = useState('idle');

  async function handleSubmit(e) {
    e.preventDefault();
    setStatus('loading');

    const form = new FormData(e.target);
    const data = Object.fromEntries(form);

    const res = await fetch('/api/contact', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    });

    if (res.ok) {
      setStatus('success');
      e.target.reset();
    } else {
      setStatus('error');
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" placeholder="Name" required />
      <input name="email" type="email" placeholder="Email" required />
      <textarea name="message" placeholder="Message" required></textarea>
      <button type="submit" disabled={status === 'loading'}>
        {status === 'loading' ? 'Sending...' : 'Send'}
      </button>
      {status === 'success' && <p>Message sent!</p>}
      {status === 'error' && <p>Something went wrong.</p>}
    </form>
  );
}

Part 3: Dynamic API Routes

Pwedeng gumamit ng dynamic segments sa API routes:

// pages/api/posts/[id].js
export default function handler(req, res) {
  const { id } = req.query;

  switch (req.method) {
    case 'GET':
      // Kunin ang post mula sa Laravel API
      return fetch(`http://localhost:8000/api/posts/${id}`)
        .then(r => r.json())
        .then(data => res.status(200).json(data))
        .catch(() => res.status(502).json({ error: 'API unavailable' }));

    case 'DELETE':
      return fetch(`http://localhost:8000/api/posts/${id}`, {
        method: 'DELETE',
        headers: { Authorization: req.headers.authorization },
      })
        .then(r => res.status(200).json({ message: 'Deleted' }))
        .catch(() => res.status(502).json({ error: 'API unavailable' }));

    default:
      res.setHeader('Allow', ['GET', 'DELETE']);
      res.status(405).json({ error: `Method ${req.method} not allowed` });
  }
}
Security Reminder Huwag mag-hardcode ng API keys sa code. Gamitin ang environment variables (next section).

Part 4: Environment Variables

Gumawa ng .env.local file sa root ng iyong Next.js project:

# .env.local
NEXT_PUBLIC_API_URL=http://localhost:8000/api
API_TOKEN=your-secret-token-here
Prefix Convention
  • NEXT_PUBLIC_ — exposed sa browser. Pwedeng gamitin sa client-side code.
  • Walang prefix — server-side lang. Hindi accessible sa browser.
Halimbawa: ang NEXT_PUBLIC_API_URL ay pwedeng gamitin sa useSWR sa browser. Ang API_TOKEN ay sa getStaticProps o API routes lamang.

Gamitin ang environment variables sa API route:

// pages/api/posts/index.js
export default async function handler(req, res) {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL;

  try {
    const response = await fetch(`${apiUrl}/posts`, {
      headers: {
        Authorization: `Bearer ${process.env.API_TOKEN}`,
      },
    });
    const data = await response.json();
    res.status(200).json(data);
  } catch (error) {
    res.status(502).json({ error: 'Failed to fetch posts' });
  }
}

Part 5: Next.js Middleware

Ang Middleware ay nagpapahintulot sa iyo na magpatakbo ng code bago ang isang request ay ma-complete — perfect para sa authentication, redirects, at rate limiting.

Gumawa ng middleware.js sa root ng iyong project:

// middleware.js
import { NextResponse } from 'next/server';

export function middleware(request) {
  const token = request.cookies.get('token');

  // Protektahan ang /dashboard routes
  if (request.nextUrl.pathname.startsWith('/dashboard')) {
    if (!token) {
      return NextResponse.redirect(new URL('/login', request.url));
    }
  }

  return NextResponse.next();
}

// Specify which routes ang maaapektuhan
export const config = {
  matcher: ['/dashboard/:path*', '/api/protected/:path*'],
};

Mini-Project: Contact Form with API Route

Gumawa ng contact form na nagse-send ng data sa pamamagitan ng Next.js API route:

  • Gumawa ng pages/api/contact.js na tumatanggap ng POST requests
  • Magdagdag ng validation — dapat may name, email, at message
  • Gumawa ng ContactForm component sa React
  • Gamitin ang fetch('/api/contact') para i-submit ang form
  • Magpakita ng loading, success, at error states
  • Bonus: Magdagdag ng API route na nagfe-fetch ng posts mula sa Laravel (proxy pattern)

Summary

  • Ang API Routes ay nasa pages/api/ — file-based routing rin
  • Gamitin ang req.method para i-handle ang iba't ibang HTTP methods
  • Ang environment variables na may NEXT_PUBLIC_ prefix ay exposed sa browser
  • Ang middleware.js ay ginagamit para sa authentication at redirects
  • Ang API Routes ay serverless — hindi kailangan ng separate backend server
